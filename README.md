JTK Learn — Web Automation Testing
=================================

Automated web UI tests for the JTK Learn demo application using Java, Selenium, TestNG, and Cucumber.

Built with **Page Object Model (POM)** pattern — each web page has a dedicated class with `@FindBy` annotations and `BasePage` reusable helpers. Designed for **high cohesion, low coupling, and reusability**.

Prerequisites
-------------

- Java 17+ installed and `JAVA_HOME` set.
- Maven 3.6+ installed and available on `PATH`.
- A supported browser installed (Chrome or Edge).

Profiles
--------

| Profile | Command | Target URL |
|---|---|---|
| `fix` (default) | `mvn test` | `/jtk-learn/` |
| `buggy` | `mvn test -P buggy` | `/jtk-learn-buggy/` |

Running the tests
-----------------

```bash
mvn clean test
```

Or use the convenience script (sets Java 23 & Maven automatically):

```powershell
.\run-tests.ps1
```

To run a single feature, use your IDE test runner on `src/test/java/runners/TestRunner.java`.

Reports
-------

- `target/cucumber-reports.html` — built-in Cucumber HTML report (single file, scenario list)
- `target/cucumber-report-html/` — advanced HTML report with overview, breakdown per feature/tag/step, and failure analysis (generated by `net.masterthought` plugin)
- `target/surefire-reports/` — raw TestNG results
